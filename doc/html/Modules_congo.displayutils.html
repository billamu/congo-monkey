Detailed Discussion

Various standalone functions relating to display and image handling. Includes resource handling (with Texture Packer support), display resolution checks, cursor hide/show, screen capture, and so on. Some have platform-specific native code.

Global Documentation

Global CONGO_DEVICE_CONTENT_SCALE : Float

Internal. Stores the current device scale factor. Don't set this manually, use EnableCongoRetina()

Function Documentation

Function CongoFontLoader : AngelFont ( fontname:String )

Similar to CongoResourceLoader, this function loads an sd/hd/xhd Angelfont font resource.

Data files for the hd font should have the hd extension, and so on. The fontname argument should be the regular (sd) font name without extension.

Texturecache is not used since AngelFont caches images itself.

If using bmGlyph, export in 'Sparrow' xml format.

Function CongoResourceLoader : Image ( imagename:String, flags:Int=image.defaultflags )

Main resource loading funtion.

This function loads a resource via the shared Texturecache, using sd, hd or xhd resolution versions depending on current settings. If a file does not exist, an error will be generated. Note, you should not mix sd/hd/xhd resources, use a consistent set.

Function DrawBox : Void ( r:Rect )

Simple rectangle drawing routine, used for debug drawing of bounding rects etc.

Function EnableCongoRetina : Void ()

Enables iOS retina modes where supported. Call this early on, i.e. must be before any autofit/layout code or resource loading is performed.

Internally, calling this function will multiply subsequent resource scalers by 2.0 if the display is Retina.

The Monkey iOS retina config setting must also be set to True for this to work correctly.

Function GetContentScaleFactor : Float ()

[iOS only] Returns the content scale factor, i.e. 'Retina' displays return 2.0. Other targets will return 1.0.

Function GetContentScaleFactor : Float ()

(dummy function for unsupported devices - returns 1.0).

Function GetFullResourceName : String ( file:String )

Handy function which returns a full resource name with the correct sd/hd/xhd folder and file extension added.

Used by CongoImageLoader and CongoFontLoader, but can be used to obtain any resource name if it follows the correct naming convention.

Function GrabScreenShot : Image ()

Returns a new Image containing a screen shot of device display. Monkey/mojo requires that this is called from within the render loop.

Note, the Image may be large, and has the dimensions of the device in pixels.

Function ImageFromTP : Image ( tpimg:Image, dataName:String, imageName:String )

Function which returns an Image grabbed from a TexturePacker atlas (exported in Generic xml format). The Image is stored in the shared Texturecache for future use (to avoid processing the xml each time). Will throw exception if no image is loaded.

Used by Sprite class LoadFromTP(), but can also be used standalone. Image will be Null if a name match cannot be found.

Use the regular (sd) resource name for the xml and image names. Note, all names are case-sensitive when searching the xml for names.

Dev note about 'GrabImage': this is not inefficient since a grabbed image and its source image share the same surface texture in mojo - no copying occurs. See http://www.monkeycoder.co.nz/Community/posts.php?topic=1934 and http://www.monkeycoder.co.nz/Community/posts.php?topic=4176.

Function IsUsingHDResources : Bool ()

Returns True if the current settings and display size support hd (double-size) resources. See CongoSettings to change the display size settings.

Generally you don't need to use this manually, since CongoImageLoader() and the Sprite class deal with it automatically. However, if you are doing some custom resource handling or other graphics code you may need to use this.

Function IsUsingXHDResources : Bool ()

Returns True if the current settings and display size support xhd (4x-size) resources. See CongoSettings to change the display size settings.

Generally you don't need to use this manually, since CongoImageLoader() and the Sprite class deal with it automatically. However, if you are doing some custom resource handling or other graphics code you may need to use this.
