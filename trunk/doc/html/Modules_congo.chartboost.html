Detailed Discussion

Chartboost ad wrapper (iOS and Android only). Requires Chartboost sdk to be setup correctly. Register new apps on the chartboost dashboard -- each target app gets its own id and sig.

** THIS CLASS IS UNDER DEVELOPMENT ** - it is useable but has limited functionality (the full api is not implemented). Currently does connection and some of the ad banner units. Callbacks/listeners not implemented.

Note - set Test Mode to Enabled via web dashboard. It reverts to Disabled every 20 attempts.

iOS: Drag the chartboost folder into the generate Xcode project. Go to Build Phases section in Xcode and add the required frameworks (see docs), incl AdSupport (weak link for pre-iOS6 support). Note that chartboost docs state that StartSession should be used in applicationDidBecomeActive, ie whenever app becomes active.

Android: See the Chartboost help pages for the required permissions to add to AndroidManifest.xml. The copy of androidmanifest.xml in the build folder can get overwritten, may be better to change the main template version instead.

Copy chartboost.jar and chartboost.jar.properties files to your ./build/android/libs folder.

Then, edit the AndroidGame class in /targets/android/modules/native/androidgame.java to be:

import com.chartboost.sdk.*; // NEW class AndroidGame extends Activity{

private Chartboost cb; // ADD THIS // ...

-- and add the extra chartboost code for onStart(), onStop(), onDestroy() and onBackPressed(), as described at https://help.chartboost.com/documentation/android.
